# 设置持续收集所有设备的日志<a name="ZH-CN_TOPIC_0000002583332490"></a>

## 功能说明<a name="section510993322119"></a>

## 命令格式<a name="section1461145662117"></a>

`npu-smi set -t sys-log-dump -s level -f path`

## 使用指南<a name="section13144102203217"></a>

- 若开启日志落盘持久化功能并设置持续收集所有设备的日志，则重启系统后会自动持续收集所有设备的日志。开启日志落盘持久化功能操作，详细参见[设置日志落盘持久化功能开关模式](设置日志落盘持久化功能开关模式.md)。
- 日志存放路径中，每个设备下都有一个时间戳文件用来保存持续收集的日志，该日志文件为当前时间戳文件。当前时间戳文件的日志容量或日志数量达到最大限制后，新日志将覆盖最早的日志。档位为1对应单个NPU设备当前时间戳文件存储日志最大容量约为100 MB，每增加1档对应单个NPU设备的当前时间戳文件存储日志最大容量增加约100 MB。
- 重启设备或重新执行当前命令，当前时间戳文件会变成历史时间戳文件。当历史时间戳文件的日志容量或日志数量达到最大限制后，会删除最早的历史时间戳文件直到历史时间戳文件的日志容量和日志数量同时满足限制。档位为1时对应单个NPU设备历史时间戳文件存储日志最大容量为200 MB，每增加1档对应单个NPU设备的历史时间戳文件存储日志最大容量增加200 MB。同时，系统最多保存99份历史时间戳文件。
- 档位为1对应单个NPU设备需要300 MB的日志存储空间，每增加1档对应单个NPU设备所需的存储空间增加300 MB，用户可根据实际设备数量和日志落盘档位计算所需的日志存储空间，并确保日志存放路径下有当前所需的存储空间。
- 如果设置持续收集所有设备的日志，升级（或覆盖安装）驱动时会先关闭日志收集任务，升级（或覆盖安装）完成后，会自动重新开启日志收集任务。

## 使用示例<a name="section614617253216"></a>

设置将所有设备的日志持续收集至/home/tmp路径下，日志容量档位为3。

```bash
npu-smi set -t sys-log-dump -s 3 -f /home/tmp
```
